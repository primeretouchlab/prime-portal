Part A — Google Drive OAuth
One-time setup · ~10 minutes
What this unlocks: When you add a client, Drive folders auto-create (Test Retouch, Selects, Review, Final Delivery, Contract). All links auto-fill into their portal. Your client data also auto-backs up to Drive after every save.
1
Go to Google Cloud Console
console.cloud.google.com
Sign in with the Google account that owns your Drive. Accept any terms if prompted.
2
Create a new project
Click the project dropdown (top left) → New Project → Name it Prime Retouch Lab → Create. Select it from the dropdown once created.
3
Enable Google Drive API
Left menu → APIs & Services → Library → Search Google Drive API → Click it → Click blue Enable.
4
Configure OAuth Consent Screen
Left menu → APIs & Services → OAuth consent screen → Select External → Create.

Fill in: App name = Prime Retouch Lab, support email = your Gmail, developer email = your Gmail. Click Save and Continue through all remaining screens without changing anything. Back to Dashboard when done.
5
Create OAuth 2.0 Client ID
Left menu → APIs & Services → Credentials → + Create Credentials → OAuth client ID.

Application type: Web application
Name: Prime Portal

Under Authorized JavaScript origins → Add URI:
https://primeretouchlab.github.io
Also add:
http://localhost
Leave Authorized redirect URIs completely empty.

Click Create. Copy your Client ID — it looks like:
123456789-abcdefgh.apps.googleusercontent.com
6
Paste your Client ID here
Not set
7
Connect in Studio tab
Go to Studio tab → tap the gold Connect banner. Google sign-in opens. Sign in → click Allow. Banner turns green. Done — all future clients auto-get Drive folders.
⚠️ Google may show "This app isn't verified." That's normal for your own apps. Click Advanced → Go to Prime Retouch Lab (unsafe). It's safe — it's accessing your own Drive.
Part B — New GitHub Repo
If you want a dedicated repo for the portal
1
Create the repo
github.com → + → New repository
Name: prime-portal · Set Public · Tick Add README → Create.
2
Upload files
Add files → Upload files. Drag in:
prime_portal.html
manifest.json
sw.js
icon-192.png
icon-512.png
Commit changes.
3
Enable GitHub Pages
Repo Settings → Pages (sidebar) → Source: Deploy from a branch → main → /(root) → Save.

Wait 2 minutes. Your portal is live at:
https://primeretouchlab.github.io/prime-portal/prime_portal.html
4
Install on phone
Android: Open URL in Chrome → three dots → Add to Home Screen.
iPhone: Open URL in Safari → Share → Add to Home Screen.
Updating Files on GitHub
When you get an updated file — go to your repo → Add files → Upload files → drag the new file in. Because the filename matches, GitHub replaces it automatically. Commit and it's live within 2 minutes.
How Notifications Work
When you mark a batch as Waiting Client (you uploaded review previews) or a batch delivery link is added — a red notification badge appears on the Portal tab and on the client card so you know there are files waiting. On the client portal, those batches get a pulsing NEW badge so the client immediately sees new files. To clear a notification, mark the batch status as Revision, Delivered, or tick it resolved.
